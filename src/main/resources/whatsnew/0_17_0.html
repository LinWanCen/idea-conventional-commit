New tokens Providers

Here we are with another feature-packed update!

The feedback issue had some success. Thanks to Gerrit-K suggestion we have now replaced the old Default - VCS Provider with a Recently used one and a VCS one.

The Recently used Provider reads tokens from the recent commit messages cache, and if it doesn't find enough tokens it falls back to the active VCS repositories filtering the commits made by yourself.
The VCS Provider reads tokens strictly from VCS repositories, without any filter so that you can pick whatever a friend or colleague wrote down.

Don't worry about performance as what's needed is cached at the appropriate time to offer a snappy completion experience. Beware reading from VCS repositories is still experimental, so if you encounter issues you're able to disable it by unchecking the 

com.github.lppedd.cc.providers.vcs

Registry key.


Be sure to keep the feedbacks coming by commenting on the GitHub issue, or by contacting me at lp.edoardo(at)gmail.com

Changes in 0.17.0

Other than the new Providers, this release brings small features, a couple of fixes, and as always an extensive refactoring of the codebase.
Following is a short summary, but you can look at all the commits here.

Features:

• implemented two new tokens Providers that are able to offer Recently used tokens and VCS-extracted tokens.

• cleaned-up the commit message even when template completion is canceled (deliberately or by accident).

• included by default the Refs footer type.

Fixes:

• deduplicated completion popup's items based on Providers ordering.

• removed a memory leak when the Providers settings panel was opened without first going through the main panel.

• correctly scaled the What's New dialog tabs and icon when a user changes scaling value while the dialog is displayed.

• correctly calculated the checkbox column width on row insertion when the Co-authors table is empty.

Refactorings:

• updated the UI style of the Co-authors table to reflect the changes made in 0.16.0.

• improved the SVG icons.

What's next?

As stated in the previous release changelog, I'm focusing mainly on the features we already have. However I'd like to provide a way to disable a Provider as currently this is impossible, if not by disabling plugins in case of third-party ones. And obviously I'll continue cleaning-up the code and fixing issues if required!
