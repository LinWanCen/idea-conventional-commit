0.11.0 (09/04/2020)

Seventeenth alpha release.
New feature, refactoring and fix

  • Provided a way to filter out a Provider's completion items directly from the completion's popup.



    As of now, if you change the search term, the filters are going to be reset.

    This experimental feature brought a massive code update (+ 1300 LOC), and laid the foundation for Provider's parallel execution. In fact the items are no more ordered thanks to pre-ordering Provider's, but only thanks to a weighing system, which updates them as they come in.

    Try it out! Feedback is highly appreciated, and ideas are welcomed

  • Fixed the handling of the closing parenthesis, which couldn't be inserted in certain situations
  • Aligned the behavior of overriding a footer value by completion item and by author's dialog
